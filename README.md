# 圆桌派 `roundtable-pai`

圆桌派是一个把你的“问题”变成三人讨论的智能 skill。

你只需要直接说一个问题，系统会先从人物库里挑出 10 位候选人物；你选 3 位之后，它就直接开始第一轮讨论。之后每一轮都会停下来，等你认同某位人物、沉默继续，或者自己插话。

## 适合做什么

- 决策会诊：我该不该做这件事？
- 观点判断：这个趋势到底意味着什么？
- 执行拆解：接下来具体该怎么做？
- 反思诊断：我为什么总卡在这里？
- 创意碰撞：有没有新的组合方式？

## OpenClaw 对话版怎么用

你不用记命令，直接像聊天一样说话就行：

1. 直接发一个问题。
2. 系统给你 10 位候选人物。
3. 你回 3 位人物名字或序号。
4. 系统直接开始第一轮讨论。
5. 每一轮结束后，你可以：
   - 回 `A` 认同本轮第一位人物
   - 回 `B` 认同本轮第二位人物
   - 回 `C` 认同本轮第三位人物
   - 回 `D` 沉默，让讨论继续
   - 回 `E` 表示“我有另外的话要说”
   - 或者直接说一句你的看法，系统会把它当作 `E` 处理，再送进下一轮

## 目录结构

- `SKILL.md`：skill 主入口
- `scripts/roundtable_controller.py`：自然语言单入口控制器
- `references/`：讨论规则、输出规则、人物 persona
- `data/`：人物池与 registry
- `runtime/roundtable_state.json`：运行态状态文件

## 人设原则

阶段 2 生成讨论时，人物表达必须直接以 `references/personas/*.md` 原文为准，尤其是：

- `人物使用规则`
- `表达DNA`
- `核心心智模型`
- `决策启发式`

`references/personas/*.md` 是上游人物目录里已经蒸馏好的快照，运行时直接以它为准。
