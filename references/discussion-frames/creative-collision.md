# Creative Collision

适用：给我新点子、还能怎么玩、怎么重新组合。

## 推荐轮数

- `5` 到 `6` 轮

## 节点结构

- `5` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `6` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 反转 / 修正 -> 结案陈词

用户可见标题只写 `第一轮` 到 `第六轮`，不要把这些节点名写给用户看。

## 开场引子

```text
你现在要先生成一段“创意碰撞型”讨论的短开场。

任务：
1. 用 1 到 3 句把三位人物请到桌前。
2. 最好由 initiator 开口，也可以是极短旁白。
3. 点出这场讨论要打破的旧框架是什么。
4. 开场要有创意感，但不要装腔或抢戏。
```

## 开题

```text
你现在要生成一场“创意碰撞型”讨论的开题轮。

任务：
1. 点出这道题真正要打破的旧框架是什么。
2. 明确这场讨论不是“多给几个普通点子”，而是要做出风格、角度或结构上的跳变。
3. 开题保持短，但要把创意边界和冲突建立起来。
```

## 立场亮相

```text
你现在要生成“创意碰撞型”的立场亮相轮。

任务：
1. 每位角色先抛一个有辨识度的初始提案。
2. 三个提案必须站在不同逻辑上，不要只是同一想法的三个变体。
3. 不允许一开始就自我审查到只剩保守点子。
```

## 交叉质询

```text
你现在要生成“创意碰撞型”的交叉质询轮。

任务：
1. 每位角色都要攻击另一位角色方案里最脆弱的一点。
2. 攻击不是情绪化否定，而是指出为什么这个方案会失败、会无聊或会不可执行。
3. 这一轮必须明显抬高创意门槛。
```

## 深水区

```text
你现在要生成“创意碰撞型”的深水区轮。

任务：
1. 把创意推进到真实约束之中，例如成本、传播门槛、执行难度、用户接受度。
2. 至少有一个方案要在受限条件下完成一次变形升级。
3. 讨论不能只停留在“酷不酷”，而要进入“能不能成立，为什么成立”。
```

## 反转 / 修正

```text
你现在要生成“创意碰撞型”的反转 / 修正轮。

任务：
1. 三位角色要在互相攻击之后做二次改造，而不是原样坚持。
2. 至少有一个方案要吸收别人的一部分思路后变得更强。
3. 这轮要让最佳方案出现明显进化，而不是简单投票。
```

## 结案陈词

```text
你现在要生成“创意碰撞型”的结案陈词轮。

任务：
1. 每位角色给出最后保留的创意判断。
2. 不再开新提案，只说最后值得试的方向。
3. 为系统总结里的实验建议自然收口。
```

## 散场过桥

```text
你现在要生成“创意碰撞型”讨论在结论字段前的短散场。

任务：
1. 用 1 到 2 句把这场碰撞先收一收。
2. 默认由 initiator 或 synthesizer 开口，谁更自然就用谁。
3. 散场要有“桌上已经碰出火花，先把话收到这”的感觉，但不要提前写实验结论。
```
