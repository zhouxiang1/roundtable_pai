# Decision Consult

适用：要不要做、要不要离职、要不要合作。

## 推荐轮数

- `4` 到 `6` 轮

## 节点结构

- `4` 轮：开题 + 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `5` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `6` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 反转 / 修正 -> 结案陈词

用户可见标题只写 `第一轮` 到 `第六轮`，不要把这些节点名写给用户看。

## 开场引子

```text
你现在要先生成一段“决策会诊型”讨论的短开场。

任务：
1. 用 1 到 3 句把三位人物请到桌前。
2. 最好由 initiator 开口，也可以是极短旁白。
3. 点出这场讨论真正要判断的，不只是“做不做”，而是“什么条件下值得做”。
4. 开场要自然，不要像系统播报。
```

## 开题

```text
你现在要生成一场“决策会诊型”讨论的开题轮。

任务：
1. 点出用户表层问法背后的核心矛盾。
2. 明确这场讨论真正要回答的，不只是“做不做”，而是“什么条件下值得做，什么条件下不值得做”。
3. 开题保持短，不抢角色正文。
```

## 立场亮相

```text
你现在要生成“决策会诊型”的立场亮相轮。

任务：
1. 让三位角色分别给出自己的初始立场，例如支持、反对、条件支持。
2. 每位角色只能提出一个最关键的判断点。
3. 三位角色的切入点必须不同。
4. 必须留下后续可争论的冲突点。
```

## 交叉质询

```text
你现在要生成“决策会诊型”的交叉质询轮。

任务：
1. 每位角色必须显式回应另一位角色的一句核心判断。
2. 每位角色必须指出对方立场里最危险的一个盲点。
3. 不能重复立场亮相轮，必须让讨论进入真正交锋。
```

## 深水区

```text
你现在要生成“决策会诊型”的深水区轮。

任务：
1. 三位角色必须讨论：如果真的按某个方向去做，代价是什么，成立条件是什么。
2. 必须进入至少两个现实维度，例如时间窗口、现金流、机会成本、最坏后果、退出代价。
3. 把问题从观点之争推进到现实代价与执行前提。
```

## 反转 / 修正

```text
你现在要生成“决策会诊型”的反转 / 修正轮。

任务：
1. 至少一位角色承认对方的一部分判断成立，并修正自己的原始立场。
2. 或者把原来的强判断改成条件判断。
3. 这轮必须让用户感觉到讨论真的被推进过，而不是角色在表演既定立场。
```

## 结案陈词

```text
你现在要生成“决策会诊型”的结案陈词轮。

任务：
1. 每位角色做最后一次立场表达。
2. 不再开新话题，只保留最后希望用户记住的判断。
3. 语气更短、更稳，让讨论自然收口，再进入系统总结。
```

## 散场过桥

```text
你现在要生成“决策会诊型”讨论在结论字段前的短散场。

任务：
1. 用 1 到 2 句把整场会诊先收住。
2. 默认由 initiator 或 synthesizer 开口，谁更自然就用谁。
3. 散场要有“先把话落到这里”的感觉，但不要提前替系统下结论。
```
