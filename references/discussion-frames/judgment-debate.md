# Judgment Debate

适用：怎么看、这个现象说明什么、这个热点应如何判断。

## 推荐轮数

- `4` 到 `6` 轮

## 节点结构

- `4` 轮：开题 + 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `5` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `6` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 反转 / 修正 -> 结案陈词

用户可见标题只写 `第一轮` 到 `第六轮`，不要把这些节点名写给用户看。

## 开场引子

```text
你现在要先生成一段“观点交锋型”讨论的短开场。

任务：
1. 用 1 到 3 句把三位人物请到桌前。
2. 最好由 initiator 开口，也可以是极短旁白。
3. 这段话要点出本场真正要争的是什么，不要只复述题目。
4. 开场要自然，有圆桌对话感，不要像系统播报。
```

## 开题

```text
你现在要生成一场“观点交锋型”讨论的开题轮。

任务：
1. 点出用户真正想判断的核心矛盾，不要复述原问题。
2. 把本题的主要争议变量抬到台面上，例如平台激励、供需结构、时间窗口、现金流、用户信任。
3. 开题保持短，像一把把靶心钉住的钉子。
```

## 立场亮相

```text
你现在要生成“观点交锋型”的立场亮相轮。

任务：
1. 每位角色必须先给一个单刀直入的判断句。
2. 不允许先讲背景，不允许讲全面性，不允许“都有道理”。
3. 三位角色对问题的判断角度必须不同。
4. 每位角色至少给出一条完整因果链：因为 X，所以 Y，所以 Z。
5. 这一轮就要带出人物自己的表达节奏，不要写成三段同款评论腔。
```

## 交叉质询

```text
你现在要生成“观点交锋型”的交叉质询轮。

任务：
1. 每位角色都必须点名回应另一位角色的一个具体判断。
2. 至少有一位角色要把争议升级到更高层框架，而不是停留在表象。
3. 不能只说“你忽略了某点”，必须明确拆掉对方哪条因果链。
4. 至少出现一次局部让步，再继续反驳。
5. 回应时保留人物自己的说话方式，例如直接拆招、讲故事、下判断、用危机语言，而不是统一书面语。
```

## 深水区

```text
你现在要生成“观点交锋型”的深水区轮。

任务：
1. 把争论推进到“如果按你的判断继续看下去，会得出什么更深的结论”。
2. 本轮结束后，读者应能看清三种判断路径的代价差异。
3. 必须落到至少两个具体维度：时间窗口、前提条件、失败模式、行动含义、机会成本。
```

## 反转 / 修正

```text
你现在要生成“观点交锋型”的反转 / 修正轮。

任务：
1. 允许角色承认对方局部成立，但必须指出更关键的误判来源。
2. 至少一位角色需要修正原始判断，或把问题重新定义。
3. 这轮结束后，用户要能看出“什么条件下你是对的，什么条件下他是对的”。
```

## 结案陈词

```text
你现在要生成“观点交锋型”的结案陈词轮。

任务：
1. 每位角色保留最后一次判断表达。
2. 不再开新争议，只给最终希望用户记住的一点。
3. 让讨论自然收口，再进入系统总结。
```

## 散场过桥

```text
你现在要生成“观点交锋型”讨论在结论字段前的短散场。

任务：
1. 用 1 到 2 句把整场话先收住。
2. 默认由 initiator 或 synthesizer 开口，谁更自然就用谁。
3. 可以有“聊到这里”“桌上的分歧已经摆明”“先把话落到这里”这种停顿感。
4. 不要在散场里提前写 6 个结果字段的内容。
```
