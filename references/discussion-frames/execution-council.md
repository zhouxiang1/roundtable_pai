# Execution Council

适用：怎么做、怎么推进、接下来拆成哪些步骤。

## 推荐轮数

- `5` 到 `7` 轮

## 节点结构

- `5` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 结案陈词
- `6` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区 -> 反转 / 修正 -> 结案陈词
- `7` 轮：开题 -> 立场亮相 -> 交叉质询 -> 深水区（一） -> 深水区（二）/ 二次升级 -> 反转 / 修正 -> 结案陈词

用户可见标题只写 `第一轮` 到 `第七轮`，不要把这些节点名写给用户看。

## 开场引子

```text
你现在要先生成一段“执行拆解型”讨论的短开场。

任务：
1. 用 1 到 3 句把三位人物请到桌前。
2. 最好由 initiator 开口，也可以是极短旁白。
3. 点出这场讨论不是列清单，而是要找到真正的执行靶心和顺序。
4. 开场要自然、有会诊感，不要像流程提示。
```

## 开题

```text
你现在要生成一场“执行拆解型”讨论的开题轮。

任务：
1. 点出用户表层执行问题背后的真正目标。
2. 明确这场讨论不是比谁更会列步骤，而是先确定“到底要完成什么、卡点在哪”。
3. 开题保持短，不要抢角色正文。
```

## 立场亮相

```text
你现在要生成“执行拆解型”的立场亮相轮。

任务：
1. 每位角色都要先明确真正目标是什么，不要直接进入步骤罗列。
2. 三位角色的切入点必须不同，例如目标、节奏、约束条件、资源配置。
3. 这一轮要把“该完成什么”说清，而不是把“怎么做”一次讲完。
```

## 交叉质询

```text
你现在要生成“执行拆解型”的交叉质询轮。

任务：
1. 每位角色必须回应另一位角色提出的路径或节奏判断。
2. 至少指出一个最可能导致执行失败的关键断点。
3. 讨论重点放在路径冲突，不要重复目标定义。
```

## 深水区

```text
你现在要生成“执行拆解型”的深水区轮。

任务：
1. 把争论推进到现实执行：资源限制、顺序依赖、验证成本、失败代价。
2. 至少给出一个检查点和一个止损点。
3. 让问题从“怎么做”进一步变成“什么先做、什么不能先做、哪里最容易跑偏”。
```

## 深水区（二）/ 二次升级

```text
你现在要生成“执行拆解型”的二次深水区轮。

任务：
1. 在上一轮基础上继续下潜，不要重复。
2. 重点讨论最容易被低估的约束，例如时间窗口、团队负荷、协同成本、验证顺序。
3. 这一轮要把方案逼到真正可执行，而不是停留在聪明建议。
```

## 反转 / 修正

```text
你现在要生成“执行拆解型”的反转 / 修正轮。

任务：
1. 至少一位角色修正自己原本的推进顺序、资源假设或优先级判断。
2. 让更稳的方案替代更好听但更危险的方案。
3. 把执行讨论从“听起来对”推进到“真能落地”。
```

## 结案陈词

```text
你现在要生成“执行拆解型”的结案陈词轮。

任务：
1. 每位角色只保留最后想让用户执行的一条关键判断。
2. 不再开新路径，只对最终方案做定锚。
3. 为系统总结留出自然收口。
```

## 散场过桥

```text
你现在要生成“执行拆解型”讨论在结论字段前的短散场。

任务：
1. 用 1 到 2 句把整场拆解先落稳。
2. 默认由 initiator 或 synthesizer 开口，谁更自然就用谁。
3. 要有“方案先收到这”的感觉，但不要把检查点和动作项提前写成总结。
```
