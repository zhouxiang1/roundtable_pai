# Quality Guard

生成完整讨论后，必须做一次质量复检。优先局部重生，不整篇推倒重来。

## 检查项

1. 第二轮开始是否存在显式回应，还是三人平行发言？
2. 每一轮是否有推进，还是只是换个说法重复？
3. 三位角色的风格与关注点是否明显不同？
4. 是否出现“这取决于”“需要结合实际情况”“每种观点都有道理”之类的安全废话主导输出？
5. 最终结论是否真的来自前文讨论？
6. 是否存在明显空洞轮次？
7. 是否把后台字段或后台过程暴露给了用户？
8. 如果是新问题，候选池是否和上一题重复过高？
9. 每一轮是否真的使用了用户题目里的关键变量，而不是空泛讨论？
10. 第二轮是否真正拆了对方逻辑，而不只是换个措辞表达不同意？
11. 深水区是否真的给出了条件、代价、时间窗口、失败模式、风险、边界中的至少一部分？
12. 系统总结前，是否存在结案陈词或等效的自然收口？
13. 整场讨论里是否出现了局势变化，例如立场修正、问题重定义、争议升级、核心矛盾迁移？
14. 旁白是否过量，是否抢了角色的戏？
15. 第一轮之前，是否有一段自然的短开场，而不是直接硬切进发言？
16. 结论字段之前，是否有一段自然的短散场，而不是上一句刚说完就直接转总结？
17. 如果 `narration_enabled=true`，是否至少真的出现了 1 到 2 条旁白？
18. 去掉人物名字后，是否仍然能大致猜出是谁在说话？
19. 三位角色是否落入同一种书面中文腔调，导致人物声音被压平？
20. 强风格人物是否真的保留了他的表达节奏，而不只是保留观点方向？
21. 强风格人物的发言，是否真的直接兑现了 persona 原文里的“句式 / 词汇 / 节奏 / 幽默 / 特有说法”？

## 输出模板

```text
输出：
- quality_score（0-100）
- issues（问题列表）
- regeneration_advice（如果要重生成，应重生成哪个节点或哪一轮，以及为什么）
```

## 定点修复规则

- 候选池重复人物：按 canonical ID 去重，用下一位候选替换。
- 候选池过于同质：替换掉风格或领域最重复的人。
- 平行发言：只重生成交叉质询轮。
- 推进不足：只重生成对应的深水区轮。
- 角色味不明显：回读 3 份 persona 原文里的“人物使用规则 / 表达DNA”，再重生成第一轮。
- 结论空洞：只重生成收束段。
- 泄露后台信息：只重写用户可见包装层，不重写讨论正文。
- 候选池重复过高：只重排候选池，不影响已存在的讨论正文。
- 没咬住用户变量：只重生成开题或立场亮相轮。
- 没有真正交锋：只重生成交叉质询轮。
- 没有条件和代价：只重生成深水区轮。
- 开头太硬：只重生成开场。
- 没有自然收口：只重生成结案陈词轮。
- 缺少散场动作：只重生成散场。
- 没有明显转折但本题应更深：补生或重生反转 / 修正轮。
- `narration_enabled=true` 但没有旁白：补生 1 到 2 条旁白。
- 人物声音太淡：不要继续调摘要卡，直接回读对应角色 persona 原文里的“人物使用规则 / 表达DNA”，再只重生该角色在失真轮次里的发言。
- 旁白过量：只删改旁白，不动角色正文。

## 推荐阈值

- `quality_score < 85`：必须局部重生，不直接交付给用户。
- `quality_score >= 85`：才可以直接交付。
